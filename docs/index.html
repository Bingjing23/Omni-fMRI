Quick Start

Process a raw fMRI volume and extract Omni-fMRI features.

The checkpoint is hosted on Hugging Face; keep the downloaded file at pretrain_checkpoint/checkpoint.pth. The smoke-test script and notebook below create a tiny synthetic NIfTI first, so a new user can verify the full preprocessing-to-feature path before preparing ADNI, HCP, ABIDE, or other benchmark data.

1. Install

Clone the repository and fetch the HF checkpoint.

git clone https://github.com/OneMore1/Omni-fMRI.git
cd Omni-fMRI
conda create -n omnifmri python=3.11 -y
conda activate omnifmri
pip install -r requirements.txt
pip install -U huggingface_hub

mkdir -p pretrain_checkpoint
huggingface-cli download OneMore1/Omni-fMRI checkpoint.pth \
  --local-dir pretrain_checkpoint
2. Preprocess

Convert raw NIfTI files into 96 x 96 x 96 x 40 NPZ segments.

python data_preparation/preprocessing.py \
  --input_dir examples/raw_nifti \
  --output_dir outputs/demo_npz \
  --pattern .nii.gz \
  --target_shape 96 96 96 \
  --segment_length 40
3. Extract

Run the frozen encoder and save feature tokens.

python extract_feat.py outputs/demo_npz \
  --checkpoint pretrain_checkpoint/checkpoint.pth \
  --output-dir outputs/demo_tokens \
  --device cuda:0 \
  --overwrite

Abstract

Voxel-level pretraining without an atlas bottleneck.

Self-supervised fMRI foundation models have improved transfer, but most still compress dense voxel signals into predefined ROI parcellations. Omni-fMRI removes that atlas dependency by operating directly on voxel-level time series while using dynamic patching to keep large-scale pretraining tractable.

Atlas-free inputs

The encoder works on voxel-level fMRI volumes rather than predefined regional summaries, preserving fine-grained spatial information and avoiding atlas-induced bias.

Dynamic patching

Background regions are pruned, simple foreground regions are merged into larger patches, and complex regions keep finer resolution for efficient global modeling.

Reproducible benchmark

The paper standardizes datasets, splits, and tasks for resting-state and task-based fMRI evaluation across diagnosis, demography, retrieval, emotion, and state prediction.

Method

Dynamic patches make voxel-level ViT pretraining practical.

Instead of forcing a dense regular grid into a hierarchical backbone, Omni-fMRI tokenizes the volume according to local spatiotemporal complexity. This reduces the encoder sequence from roughly 14K tokens to about 4.3K tokens while keeping global self-attention over informative brain tissue.

1

Allocate content-adaptive patches

Prune non-brain background and assign patch scale by local signal variability.

2

Project mixed-scale tokens

Use dual-path multi-scale embedding with ZeroMLP for stable coarse-to-fine learning.

3

Reconstruct by scale

Route masked-autoencoding targets to scale-aware prediction heads.

Results

Strong transfer across resting-state and task-based fMRI.

95.54%

HCP sex classification ACC

Omni-fMRI leads the benchmark table while also achieving 95.05 F1 on HCP sex classification.

88.77%

ADNI AD diagnosis ACC

The model reaches the best reported ADNI Alzheimer's disease diagnosis accuracy in the paper's main comparison.

23.08%

Top-10 image retrieval

On 100-way image retrieval, Omni-fMRI improves over NeuroSTORM, SwiFT, and TFF in Top-10, Top-5, and Top-1 accuracy.

45.29%

HCP task accuracy at 10% data

Few-shot evaluation shows the model preserves strong task classification performance under severe data reduction.

Benchmark summary

Linear probing stays competitive.

With a frozen encoder and a lightweight linear head, Omni-fMRI often surpasses fine-tuned baselines and shows a smaller ADNI AD degradation than NeuroSTORM LP.

Efficiency summary

Scalability is part of the method.

The paper reports a 157M-parameter model pretrained for 35 epochs in about 32 hours on 4 NVIDIA A10G GPUs, enabled by dynamic token reduction.
